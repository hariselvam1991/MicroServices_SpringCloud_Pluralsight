OAuth2.0
	Protocol for conveying authorization
	Provides authorization flow for various clients
	Obtain limited access to user accounts
	Separates the idea of user and client
	Access token carried more than identity
	Not an authentication scheme
	
Actors in an OAuth2.0 Scenario
	Resource Owner - Entity grants access to a resource. Usually, you!
	Resource Server - Server hosting the protected resource.
	Client - App that's making prtected resource requests on behalf of resource owner.
	Authorization Server - Server issuing acccess tokens to clients.
